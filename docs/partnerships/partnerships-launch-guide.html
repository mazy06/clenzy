1. Vue d'ensemble & objectifs

Ce guide accompagne le lancement structuré des partenariats commerciaux et techniques entre Clenzy et l'ensemble des fournisseurs déjà intégrés dans le PMS. À ce stade, le code de connexion a été développé pour 50+ services répartis sur 17 catégories. Il reste à transformer ces intégrations techniques en partenariats commerciaux officiels.

Pourquoi le faire maintenant

  1. Crédibilité commerciale : un host qui hésite entre Clenzy et un concurrent vérifiera les partenariats officiels (badges, étude de cas, mentions sur le site du fournisseur). Aujourd'hui Clenzy a l'intégration technique mais pas la visibilité.
  2. Revenus d'affiliation : la plupart des SaaS B2B SaaS offrent 10-30 % de commission sur les referrals. Volume estimé à terme : 50-200 €/mois par client référé selon le tier.
  3. Tarifs préférentiels : statut Partner ouvre des discounts (10-30 %) à proposer aux clients Clenzy, créant un avantage tarifaire net.
  4. Co-marketing : webinaires, études de cas, listing dans le marketplace du partenaire = canal de leads gratuit.
  5. Accès anticipé : roadmap produit du partenaire, beta features, support prioritaire.

3. Sales kit Clenzy

3.1 — Les 10 points clés à mémoriser

  1. Clenzy est un PMS multi-tenant SaaS pour la location courte durée, opéré depuis la France.
  2. Marchés ciblés : France métropolitaine, Maroc, Arabie Saoudite — corridor euro-méditerranéen unique sur le marché.
  3. Stack moderne : Java 21 / Spring Boot 3.2 / React 18 / PostgreSQL 16 / Kafka — design Strategy + Registry pour les intégrations.
  4. 17 catégories d'intégrations déjà branchées (signature, pricing, compta, KYC, channel managers, OTAs, etc.).
  5. 3 langues natives : français, anglais, arabe (incluant RTL pour le booking engine).
  6. Conformité légale forte : RGPD, fiche police FR, chiffrement AES-256-GCM, NF compliance comptable, multi-juridictions.
  7. Booking engine SDK embarquable : widget JS intégrable sur n'importe quel site host, multi-langues + RTL.
  8. Multi-rails de paiement : Stripe Europe, PayTabs KSA, CMI Maroc, payouts Stripe Connect + SEPA + Wise (roadmap).
  9. Différenciation clé : seul PMS occidental avec intégration native compliance MA (DGSN) et KSA (Absher) en roadmap.
  10. Croissance : phase commerciale active, cible 200+ propriétés sous gestion en S2 2026.

3.3 — Argumentaires différenciants par type de partenaire

Pour les channel managers (Hostaway, SiteMinder, Rentals United)

Insister sur la complémentarité PMS + CM. Clenzy gère le booking direct + la résidence + l'ops, le channel manager gère la distribution OTA. Pas de concurrence, synergie. Volume prévisible 100+ propriétés.

Pour les SaaS pricing (PriceLabs, Beyond, Wheelhouse)

Marché STR encore sous-équipé en pricing dynamique en France et MA. Clenzy peut être un canal d'éducation et de distribution majeur. Co-marketing : étude de cas conjointe sur l'impact RevPAR.

Pour les KYC (Sumsub, Veriff, Onfido)

Réservations à fort risque (KSA, dépôt de garantie élevé, événements). Clenzy peut intégrer le KYC dans le flow booking engine natif — différenciation forte vs. PMS concurrents qui sous-traitent à Airbnb/Booking.

Pour les compliance (Chekin, etc.)

Marché FR : obligation légale fiche police = pain point connu de tous les hosts. Clenzy automatise cette obligation, Chekin est le maillon technique. Volume prévisible direct.

Pour les marketplaces activités (GetYourGuide, Klook, Viator)

Audience captive de guests qualifiés. Cross-sell activités via le booking engine = revenue affiliation. Clenzy peut offrir l'audience, le partenaire offre le produit.

Pour les hardware (Nuki, Minut, Igloohome, TTLock)

Distribution + intégration. Clenzy peut proposer le hardware en bundle avec son abonnement PMS, créant un revenu hardware + une fidélisation. Co-promotion sur les salons spécialisés.

16.5 — Avertissements importants

À vérifier au premier contact

Les emails listés dans ce guide sont vérifiés à la date de rédaction. Certains peuvent changer (équipes partnerships qui bougent, refontes de site). En cas de bounce, consulter le site partenaires à jour ou rechercher LinkedIn « Partnership manager [Provider] ».

Localisation des emails par marché

Pour les partenaires FR (Pennylane, Yousign, Universign), envoyer les emails en français. Pour les partenaires globaux ou MENA, l'anglais reste la langue par défaut. Adapter le ton selon la culture business locale.

Bonne pratique : LinkedIn en parallèle

Pour chaque email envoyé à partners@xxx.com, identifier également le Partnership Manager sur LinkedIn et envoyer une invite avec un mot personnalisé. Le taux de réponse double quand le contact est triangulé email + LinkedIn.
